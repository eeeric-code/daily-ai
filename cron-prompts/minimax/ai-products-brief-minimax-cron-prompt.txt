> 之前只改 portal-template.html 会导致:minimax 产业 07:00 跑完 → 用户看首页仍是最旧的 → 必须等 09:00 gpt cron 才能刷新。
> 现在**两个地方都改**:portal-template.html(源)+ repo/index.html(部署版,保证立即生效)。

**PowerShell 示例**:
```powershell
$deployedIndex = "D:\ECNU\其他\每日AI推送\repo\index.html"
$repoMinimaxProductsPath = "D:\ECNU\其他\每日AI推送\repo\ai-products\minimax"
$content = Get-Content $deployedIndex -Encoding UTF8 -Raw

# 计算 minimax 产业 slot 实际期数
$minimaxProductsCount = (Get-ChildItem -Path $repoMinimaxProductsPath -Directory -ErrorAction SilentlyContinue |
    Where-Object { $_.Name -match '^\d{4}-\d{2}-\d{2}$' }).Count

# 字段 A:minimax 产业 最新日期(限定在 "minimax · global products" card 块内)
$content = $content -replace '(<span class="model-name">minimax · global products</span>[\s\S]*?<span>最新[：:]?)\d{4}-\d{2}-\d{2}', "`$1$date"

# 字段 B:minimax 产业 收录数(倒装"已收录 N 期",限定在 "minimax · global products" card 块内)
$content = $content -replace '(<span class="model-name">minimax · global products</span>[\s\S]*?<span>已收录 ?)\d+( ?期</span>)', "`$1$minimaxProductsCount`$2"

# 字段 C:minimax 产业 阅读最新链接(限定在 "minimax · global products" card 块内,path 含 ai-products/minimax/)
$content = $content -replace '(<span class="model-name">minimax · global products</span>[\s\S]*?href="\./ai-products/minimax/)\d{4}-\d{2}-\d{2}(/)"', "`$1$date`$2"

Set-Content $deployedIndex -Value $content -Encoding UTF8 -NoNewline
Write-Host "✓ patched repo/index.html minimax 产业卡片: latest=$date count=$minimaxProductsCount"
```

**关键安全点**:
- `[\s\S]*?` 限定在 model-name 块内,绝不跨过下一个 `<section class="model-card">`
- model-name 严格匹配 `>minimax · global products<`,不匹配 `>minimax<`(那是综合 card)
- 字段 B 倒装"已收录 N 期",gpt card 用的"N 期已收录"不匹配此 regex
- 字段 C path 限定 `ai-products/minimax/`,不匹配 `ai-daily/minimax/`

**gpt 综合 / gpt 产业 / minimax 综合 card 完全不动**(只动 minimax 产业 card)。

# 四、minimax 产业 cron 与其他 cron 协调规则
- gpt 产品产业 cron:每天 09:00,部署到 `repo/ai-products/gpt/$DATE/`(gpt slot) — **尚未建立为 cron,目前手动触发**
- minimax 产业 cron(本编排提示):每天 07:00,部署到 `repo/ai-products/minimax/$DATE/`(minimax slot)
- minimax 综合 cron:每天 08:30,部署到 `repo/ai-daily/minimax/$DATE/`
- gpt 综合 cron:每天 09:00,部署到 `repo/ai-daily/$DATE/`(gpt slot)
- 4 个 cron **文件路径不重叠**(ai-products vs ai-daily,gpt vs minimax 互不交叉)
- **时间错峰**:07:00 / 07:30(minimax 产业) → 08:30 / 09:00(minimax+gpt 综合) — git push 完全不撞车
- minimax 产业 **不读**:
  - `.cron-state-products-gpt.json`(gpt 产业的)
  - `.cron-state-minimax.json`(minimax 综合的)
  - `.cron-state.json` 或 `.cron-state-gpt.json`(gpt 综合的)
- minimax 产业 **不写**:
  - `repo/ai-daily/...`(综合简报)
  - `repo/ai-products/gpt/...`(gpt 产业 slot)
  - `repo/briefs.html`(gpt 维护)
  - `repo/example/...`(只读)
