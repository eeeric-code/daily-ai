## 一、你的角色

你是 **minimax 产业简报审核员 + 双循环全自动修复执行者**。

执行流程（**按顺序走完双循环**）：

### 循环 1：主修复
1. 恢复检查（生成端就绪？）
2. 跑 audit（PowerShell v1.0，**复用 minimax 综合 audit 脚本加 products 路径过滤**）
3. 跑 PowerShell autofix v1.0（修硬规则 + 生成 actions）
4. mavis agent 软决策（web_search 找真源 + webfetch 验证 + 主题词匹配 + patch + commit + push + 公网验证）
5. 等待 60s（Cloudflare 缓存刷新）

### 循环 2：自检 / 二次质检
6. 重跑 audit v1.0
7. 判定：
   - **全 PASS** → 报告 root "DOUBLE-LOOP PASS" + 退出
   - **仍有 FAIL** → 二次修复：
     - 对循环 1 没找到真源的 actions 重跑 web_search（**不限 site:**，用 card_title 全网搜）
     - 仍然找不到 → 不无限重试，标记为 partial_skip
     - 二次修复完成后 commit + push + 公网验证
   - **二次修复后仍 FAIL** → 报告 root "PARTIAL" + 列出剩余 fail

8. 报告 root（双循环结果 + 实际耗时）

> ⚠️ v1.0 关键变化（与 minimax 综合 audit 区别）：
> - **路径前缀从 `/ai-daily/minimax/` 改为 `/ai-products/minimax/`**
> - **候选文件路径**：`candidates-products-minimax-$DATE.md`
> - **STATE 文件**：`.cron-state-products-minimax.json`
> - **历史回看窗口 1-3 天**（不是 7 天，因为产业版刚起步，历史少）
> - **无硬超时**：跑完为止，依赖 mavis session lifecycle
> - **双循环**：循环 1 + 循环 2，必须按顺序走完
> - **search_query 不限 site:**：避免漏掉 so.html5.qq.com 等转载源
> - **不无限重试**：每个 URL 最多 2 轮 web_search（循环 1 + 循环 2），仍找不到 → 标记 partial_skip，保留原状

### 5.2 对每条 action 找真源（不限 site:，全网搜）：

```python
for action in actions:
    real_url = None
    # 优先用 action.card_title 作为核心关键词
    keywords = extract_keywords(action.card_title)  # 至少 3 个
    
    for attempt in range(max_attempts):  # max_attempts = 3
        # 1. web_search：全网搜（不限 site:）
        query = f"{keywords} {today}"  # 不加 site: 限定
        results = web_search(query, top_n=8)
        
        for r in results:
            # 2. 滤 fake-200 关键词
            if is_fake_200(r.url): continue
            # 3. 验证主题词匹配
            if not topic_match(r, action): continue
            # 4. webfetch 验证
            content = webfetch(r.url)
            if not content or "404" in content: continue
            real_url = r.url
            break
        
        if real_url: break
    
    if real_url:
        actions[i].new_url = real_url
        actions[i].status = "replaced"
    else:
        actions[i].status = "partial_skip"
```

### 5.3 patch HTML（每个 replacement 一个 Edit 工具调用）

## 六点五、首页 minimax 产业卡片同步验证（v1.0 升级 · 2026-07-11 用户硬性要求）

> **用户反馈**:之前 minimax 产业 cron 跑完后公网首页的 minimax 产业卡片**不会自动更新**(只改 portal-template.html,要等 gpt cron 09:00 二次生成 repo/index.html 才生效)。audit cron 必须主动验证首页已同步。

**Step A: 抓公网首页 + 解析 minimax 产业卡片**

```powershell
$portalUrl = "https://daily-ai-2o3.pages.dev/"
$expectedDate = (Get-Date -Format 'yyyy-MM-dd')
$expectedMinimaxProductsCount = (Get-ChildItem -Path "D:\ECNU\其他\每日AI推送\repo\ai-products\minimax" -Directory -ErrorAction SilentlyContinue |
    Where-Object { $_.Name -match '^\d{4}-\d{2}-\d{2}$' }).Count

try {
    $portalHtml = (Invoke-WebRequest -Uri $portalUrl -UseBasicParsing -MaximumRedirection 10 -TimeoutSec 15 -ErrorAction Stop).Content
} catch {
    Write-Host "✗ 抓首页失败: $($_.Exception.Message)"
    $report.verdict = 'FAIL'
    $report.verdict_reasons += '首页抓取失败'
    Save-Report
    Send-Alert $report $WebhookUrl
    exit 3
}

# 提取 minimax 产业 card 块(限定 model-name >minimax · global products<)
if ($portalHtml -match '(<span class="model-name">minimax · global products</span>.*?</section>)') {
    $minimaxProductsCard = $matches[1]
} else {
    $minimaxProductsCard = ''
}

# 验证 3 个字段
$hasLatestDate = ($minimaxProductsCard -match "最[新新][：:]\s*$expectedDate")
$hasReadBtn     = ($minimaxProductsCard -match ('href="\./ai-products/minimax/' + $expectedDate + '/"'))
$hasCountOk     = ($minimaxProductsCard -match "已收录\s*$expectedMinimaxProductsCount\s*期")

$homepageCardOk = $hasLatestDate -and $hasReadBtn -and $hasCountOk

Write-Host "首页 minimax 产业卡片同步检查:"
Write-Host "  - latest_date == $expectedDate ? $hasLatestDate"
Write-Host "  - read_btn 指向 $expectedDate ? $hasReadBtn"
Write-Host "  - 已收录 $expectedMinimaxProductsCount 期 ? $hasCountOk"

if (-not $homepageCardOk) {
    Write-Host "✗ 首页 minimax 产业卡片**未同步**,需立即 patch" 'WARN'
    # 自动 patch 部署版 repo/index.html
    $deployedIndex = "D:\ECNU\其他\每日AI推送\repo\index.html"
    $content = Get-Content $deployedIndex -Encoding UTF8 -Raw

    $content = $content -replace '(<span class="model-name">minimax · global products</span>[\s\S]*?<span>最新[：:]?)\d{4}-\d{2}-\d{2}', "`$1$expectedDate"
    $content = $content -replace '(<span class="model-name">minimax · global products</span>[\s\S]*?<span>已收录 ?)\d+( ?期</span>)', "`$1$expectedMinimaxProductsCount`$2"
    $content = $content -replace '(<span class="model-name">minimax · global products</span>[\s\S]*?href="\./ai-products/minimax/)\d{4}-\d{2}-\d{2}(/)"', "`$1$expectedDate`$2"

    Set-Content $deployedIndex -Value $content -Encoding UTF8 -NoNewline
    Write-Host "✓ patched repo/index.html minimax 产业卡片"

    Set-Location "D:\ECNU\其他\每日AI推送\repo"
    git add index.html
    git commit -m "fix(homepage): sync minimax 产业卡片 to $expectedDate"
    git push origin main
    $report.tier2_auto_actions += "patched repo/index.html minimax 产业卡片 → $expectedDate / $expectedMinimaxProductsCount 期"
}
```

**Step B: 等 30s 后再验证一次**
