### 4.4 更新 portal-template.html 首页源(minimax 卡片部分)
**这是首页的源文件**(`D:\ECNU\其他\每日AI推送\portal-template.html`),gpt cron 每天会从这里重新生成 `repo/index.html`。
minimax 改这里的 minimax 卡片,gpt cron 重新生成时会把变更带过去。

只改 minimax card 块内 3 个字段:
- **字段 A** 最新日期: `<span>最新：YYYY-MM-DD</span>` → 今天
- **字段 B** 收录数(倒装): `<span>已收录 N 期</span>` → minimax slot 实际期数(扫描 `repo/ai-daily/minimax/` 下 YYYY-MM-DD 子目录数)
- **字段 C** 阅读最新链接: `href="./ai-daily/minimax/YYYY-MM-DD/"` → 今天

**重要**:字段 B 用倒装"已收录 N 期",**绝不**写成"N 期已收录"。原因:gpt cron §9.4 有 regex `(\d+)\s*期已收录` 会同时匹配 minimax 卡片,把 minimax 的收录数覆盖成 gpt 算的总数。用倒装避开这个 regex。

**gpt card 块完全不动**(只动 minimax card)。

**PowerShell 示例**:
```powershell
$portalPath = "D:\ECNU\其他\每日AI推送\portal-template.html"
$content = Get-Content $portalPath -Encoding UTF8 -Raw

# 字段 A:minimax 最新日期(限定在 minimax card 块内)
$content = $content -replace '(<span class="model-name">minimax</span>[\s\S]*?<span>最新：)\d{4}-\d{2}-\d{2}', "`$1$date"

# 字段 B:minimax 收录数(倒装,限定在 minimax card 块内)
$minimaxCount = (Get-ChildItem -Path "D:\ECNU\其他\每日AI推送\repo\ai-daily\minimax" -Directory -ErrorAction SilentlyContinue | Where-Object { $_.Name -match '^\d{4}-\d{2}-\d{2}$' }).Count
$content = $content -replace '(<span class="model-name">minimax</span>[\s\S]*?<span>已收录 )\d+( 期</span>)', "`$1$minimaxCount`$2"

# 字段 C:minimax 阅读最新链接(限定在 minimax card 块内)
$content = $content -replace '(<span class="model-name">minimax</span>[\s\S]*?href="\./ai-daily/minimax/)\d{4}-\d{2}-\d{2}(/)"', "`$1$date`$2"

Set-Content $portalPath -Value $content -Encoding UTF8 -NoNewline
```

### 4.4b 直接 patch 部署版 `repo/index.html` (独立更新 minimax 综合卡片,绕过 gpt cron 二次生成)

> **2026-07-11 用户明确要求**:minimax cron 跑完必须**立即**在公网首页看到 minimax 卡片更新,不能等 gpt cron 9:00 二次生成。

> 现在**两个地方都改**:portal-template.html(源)+ repo/index.html(部署版,保证立即生效)。

**PowerShell 示例**:
```powershell
$deployedIndex = "D:\ECNU\其他\每日AI推送\repo\index.html"
$repoMinimaxPath = "D:\ECNU\其他\每日AI推送\repo\ai-daily\minimax"
$content = Get-Content $deployedIndex -Encoding UTF8 -Raw

# 计算 minimax 综合 slot 实际期数(扫描 repo\ai-daily\minimax\ 下 YYYY-MM-DD 子目录数)
$minimaxCount = (Get-ChildItem -Path $repoMinimaxPath -Directory -ErrorAction SilentlyContinue |
    Where-Object { $_.Name -match '^\d{4}-\d{2}-\d{2}$' }).Count

# 字段 A:minimax 最新日期(限定在 minimax 综合 card 块内,model-name 严格匹配"minimax"排除"minimax · global products")
$content = $content -replace '(<span class="model-name">minimax</span>[\s\S]*?<span>最新[：:]?)\d{4}-\d{2}-\d{2}', "`$1$date"

# 字段 B:minimax 收录数(倒装"已收录 N 期",限定在 minimax 综合 card 块内)
$content = $content -replace '(<span class="model-name">minimax</span>[\s\S]*?<span>已收录 ?)\d+( ?期</span>)', "`$1$minimaxCount`$2"

# 字段 C:minimax 阅读最新链接(限定在 minimax 综合 card 块内,path 含 ai-daily/minimax/)
$content = $content -replace '(<span class="model-name">minimax</span>[\s\S]*?href="\./ai-daily/minimax/)\d{4}-\d{2}-\d{2}(/)"', "`$1$date`$2"

Set-Content $deployedIndex -Value $content -Encoding UTF8 -NoNewline
Write-Host "✓ patched repo/index.html minimax 综合卡片: latest=$date count=$minimaxCount"
```

**关键安全点**:
- `[\s\S]*?` 限定在 model-name 块内,绝不跨过下一个 `<section class="model-card">`
- model-name 严格匹配 `>minimax<` (排除 `>minimax · global products<`)
- 字段 B 倒装"已收录 N 期",gpt card 用的"N 期已收录"不匹配此 regex
- 字段 C path 限定 `ai-daily/minimax/`,不匹配 `ai-products/minimax/`

**gpt 综合 / gpt 产业 / minimax 产业 card 完全不动**(只动 minimax 综合 card)。
